| **两层** | 两层结构目录页json+详情页 | JSON列表 → 详情页 |

## 二、按数据源格式分类

| 数据格式 | 对应模板 |
|---|---|
| **HTML页面（XPath/CSS解析）** | 资讯两层、html目录两层、报纸三层、反爬模板 |
| **JSON接口** | json目录两层、四层翻页、百家号、搜狐号 |
| **RSS/XML** | 单层rss订阅 |
| **正则匹配** | 双层正则匹配+详情页 |
| **百度入口（代理起始）** | 百度+目录+详情页、百度+目录+文件下载 |

## 三、按特殊能力标识

| 能力标识 | 说明 | 对应模板 |
|---|---|---|
| **翻页** | 自动生成分页URL | 四层翻页+JSON目录 |
| **文件下载** | 下载PDF/附件并上传OSS | 百度+文件下载、四层翻页+文件下载 |
| **反爬** | beforeDownloader + INNER_PYTHON | 反爬脚本模板 |
| **图片内嵌** | 图片AES解密/Base64转换 | 搜狐号模版 |
| **百度起始** | 以百度URL为跳板，动态生成真实请求 | 百度+目录+详情页、百度+目录+文件下载 |
| **时间过滤** | 按时间范围筛选数据 | 百家号模版 |
| **报纸解析** | 版面号/版面信息提取 | 报纸解析模板 |
| **RSS解析** | XML item节点 + 月份英文转数字 | 单层rss订阅 |

## 四、需求描述格式

按以下格式描述需求，即可快速匹配合适模板：

> **结构层级**：几层？（单层/两层/三层/四层）
> **数据格式**：HTML页面 / JSON接口 / RSS / 其他
> **起始方式**：直接URL / 百度入口 / 动态生成URL
> **特殊需求**：翻页? 文件下载? 反爬? 图片解密? 时间过滤? 报纸版面?
> **目标站点**：网址或站点名称

**示例**：
> "两层结构，列表页是JSON格式，详情页是HTML，需要翻页，目标是XX站点"
