# Groovy 采集脚本内置能力说明

## 脚本形态与入口

采集脚本是一个 Groovy 类，必须实现接口 com.gildata.spider.base.starter.groovy.IparseScript。核心入口方法如下：

*   **ParsePageResult doParse(ParsePageRequest request)**：解析页面，产出数据、子请求、去重键、错误信息等。
    
*   **AntiPageRequest beforeDownloader(AntiPageRequest request)**：可选。下载前的反爬请求增强配置；返回 null 表示跳过反爬处理。
    

**最小骨架示例：**

import com.gildata.spider.base.starter.groovy.IparseScript  
import com.gildata.spider.base.starter.groovy.model.ParsePageRequest  
import com.gildata.spider.base.starter.groovy.model.ParsePageResult  
  
class MyScript implements IparseScript {  
  @Override  
  ParsePageResult doParse(ParsePageRequest request) {  
    def page = request.getPage()  
    def result = new ParsePageResult()  
    // TODO: 解析 page，填充 result.addDataMap / result.addChildRequest 等  
    return result  
  }  
}  

## 上下文对象

*   **ParsePageRequest**：脚本入参，包含 Page page 与 contextDataMap。
    
*   **ParsePageResult**：脚本出参，包含 dataMap、childRequests、deDupKeyList、ossFileBytes、errorList、contextDataMap 等。
    
*   **AntiPageRequest**：beforeDownloader 的返回值，用于指定反爬类型、传参、是否需要代理等。
    

**提示**：系统会在 ParsePageRequest.contextDataMap 中注入 instanceId，key 为 GroovyConstants.CONTEXT\_INSTANCE\_ID，可用于日志关联与调试。

## Request extra 常量（GroovyConstants）

这些 key 主要用于 Request.putExtra(key, value) 或 Request.getExtra(key)：

*   callProxy（常量名 CALL\_PROXY）
    
*   instanceId（常量名 CONTEXT\_INSTANCE\_ID）
    
*   customCookies（常量名 CUSTOM\_COOKIES）
    
*   customProxy（常量名 CUSTOM\_PROXY）
    
*   downloadTimeoutMs（常量名 DOWNLOAD\_TIMEOUT\_MS）
    
*   fanpaProxy（常量名 FANPA\_PROXY）
    
*   firstPage（常量名 FIRST\_PAGE）
    
*   pageReload（常量名 PAGE\_RELOAD）
    
*   postParams（常量名 POST\_PARAMS）
    
*   skipDedup（常量名 SKIP\_DEDUP）
    
*   useCustomProxy（常量名 USE\_CUSTOM\_PROXY）
    
*   useLastHeaders（常量名 USE\_LAST\_HEADERS）
    

## 安全与限制（GroovyScriptUtil）

脚本保存/执行前会做安全校验，入口为 GroovyScriptUtil.safeValidaScript。主要限制如下：

*   **禁止导入或使用的包/类**：
    
    *   java.lang.System
        
    *   java.lang.Runtime
        
    *   java.lang.ProcessBuilder
        
    *   java.io
        
    *   java.net
        
    *   java.sql
        
    *   javax.script
        
    *   groovy.lang.GroovyShell
        
*   **禁止的高危关键字/模式**：
    
    *   System\.exit
        
    *   Runtime\.getRuntime
        
    *   ProcessBuilder
        
    *   Class\.forName
        
    *   ClassLoader
        
    *   Reflection
        
    *   new\s+File
        
    *   new\s+Socket
        
    *   new\s+URL
        
    *   URLConnection
        
    *   ScriptEngine
        
    *   addTargetRequest
        

## 五、内置工具函数

这部分是 Groovy 脚本里最常用的“现成能力”。建议把它们理解为 5 类能力：

*   **ParseToolsUtils**：文本、时间、正则、HTML、链接处理
    
*   **ImageToolsUtils**：图片链接补全、转 Base64
    
*   **LogUtils**：脚本运行日志输出
    
*   **SpiderManagerUtils**：请求组装、Cookie 域判断、POST 参数标准化、反爬协作
    
*   **XPathContainsReplacer**：处理 contains(text(), ...) 这类不好直接提取的 XPath 场景
    

如果脚本里没有做静态导入，也可以显式导入工具类：

import com.gildata.spider.base.starter.groovy.util.ParseToolsUtils  
import com.gildata.spider.base.starter.groovy.util.ImageToolsUtils  
import com.gildata.spider.base.starter.groovy.util.LogUtils  
import com.gildata.spider.base.starter.groovy.util.SpiderManagerUtils  
import com.gildata.spider.base.starter.groovy.util.XPathContainsReplacer  

### 5.1 ParseToolsUtils：解析脚本里的通用瑞士军刀

ParseToolsUtils 主要用于“把原始页面内容整理成可入库字段”。常见能力如下。

#### 1）正则与文本截取

**常用方法**：

*   getByRegex(text, regex)：取第一个匹配结果
    
*   extractFirstGroup(text, regex)：取第一个分组
    
*   extractAll(input, regex)：取全部匹配
    
*   getMidTextByRegex(text, startRegex, endRegex)：取两段标记之间的内容
    
*   extractTextWithTrim(text, regex, groupIndex)：提取后自动 trim
    
*   textReplace(...)、regexReplace(...)、regexReplaceAll(...)、batchReplaceString(...)：文本替换
    

**示例**：从文章页源码中提取标题、发布时间、附件链接。

def html = request.getPage().getRawText()  
  
def title = ParseToolsUtils.extractFirstGroup(html, '<h1\[^>\]\*>(.\*?)</h1>')  
def publishTime = ParseToolsUtils.extractFirstGroup(html, '发布时间\[:：\]\\s\*(\[0-9\\- :/\]+)')  
def attachmentUrls = ParseToolsUtils.extractAll(html, 'href="(.\*?\\.(pdf|docx?|xlsx?))"')  
  
title = ParseToolsUtils.regexReplaceAll(title ?: '', '<\[^>\]+>', '')  
publishTime = ParseToolsUtils.normalizeDateTime(publishTime)  

**适用建议**：

*   页面结构不稳定、XPath 不好写时，正则提取通常更快。
    
*   extractFirstGroup 比 getByRegex 更适合明确依赖分组的场景。
    
*   对标题、正文片段做清洗时，先提取再替换，代码更清晰。
    

#### 2）时间标准化

**常用方法**：

*   formatPushTimeDateTime(dateString)
    
*   formatPushTimeDateTime(input, inputFormat)
    
*   standardDateByFormat(dataStr, format)
    
*   normalizeDateTime(input)
    
*   timestamp2StandardDate(timestamp)
    
*   smartGetPushTime(sourceTime)
    
*   isDateInRange(dateToCheckStr, startDayOffset, endDayOffset)
    
*   dateRangeCheck(targetDate, startDate, endDate, trueValue, falseValue)
    

**示例**：把不同格式的发布时间统一成标准格式，并判断是否需要入库。

def rawTime = page.xpath("//span\[@class='time'\]/text()").get()  
def pushTime = ParseToolsUtils.smartGetPushTime(rawTime)  
  
if (!ParseToolsUtils.isDateInRange(pushTime, -7, 0)) {  
    return new ParsePageResult()  
}  
  
def data = \[:\]  
data.put("pushTime", pushTime)  

**示例**：已知格式时直接显式转换。

def rawTime = "2026/03/31 18:05"  
def pushTime = ParseToolsUtils.standardDateByFormat(rawTime, "yyyy/MM/dd HH:mm")  

**适用建议**：

*   网站时间格式很杂时优先用 smartGetPushTime。
    
*   已知源格式时优先用 standardDateByFormat 或 formatPushTimeDateTime(input, inputFormat)。
    
*   dateRangeCheck 适合做“命中时间窗口后返回不同值”的表达式判断。
    

#### 3）HTML 清洗与链接补全

**常用方法**：

*   buqiHtml(html)、completeHtml(brokenHtml)：补全残缺 HTML
    
*   removeElementsByXpath(htmlContent, xpathExpressions)、xpathRemove(html, xpath)：移除无关节点
    
*   completeImageHtml(htmlContent, baseUrl, imgAtr)、convertImgRelativeToAbsolute(htmlContent, baseUrl)：把相对图片地址补成绝对地址
    
*   resolve(baseUriStr, href)、getBaseUrl(url)、convertRelativeUrlsToAbsolute(page)：链接补全
    

**示例**：清理正文中的脚本、样式和分享组件，并补全图片地址。

def contentHtml = page.xpath("//div\[@class='article-content'\]").get()  
contentHtml = ParseToolsUtils.removeElementsByXpath(  
    contentHtml,  
    "//script&&//style&&//div\[contains(@class,'share')\]"  
)  
contentHtml = ParseToolsUtils.convertImgRelativeToAbsolute(contentHtml, page.getUrl().toString())  
contentHtml = ParseToolsUtils.completeHtml(contentHtml)  

**示例**：把详情页中的相对链接补成可直接请求的 URL。

def href = page.xpath("//a\[@class='next-page'\]/@href").get()  
def nextUrl = ParseToolsUtils.resolve(page.getUrl().toString(), href)  

#### 4）JSON / JSONP / XPath 读取

**常用方法**：

*   safeGetJsonPath(item, path)：安全读取 JSON 字段
    
*   extractJsonFromJsonp(jsonp)：把 JSONP 转成 JSON 对象
    
*   getXpathText(page, xpath)：用 XPath 直接取文本
    

**示例**：接口返回是 JSONP 时，先转 JSON 再取字段。

def jsonp = page.getRawText()  
def json = ParseToolsUtils.extractJsonFromJsonp(jsonp)  
def title = ParseToolsUtils.safeGetJsonPath(json, "$.data.title")  
def author = ParseToolsUtils.safeGetJsonPath(json, "$.data.author")  

#### 5）字符串拼接与辅助函数

**常用方法**：

*   addPrefix(...)、addSuffix(...)、addPrefixAndSuffix(...)
    
*   getSplittedItem(input, index)、getSplittedLastItem(input)
    
*   getFileName(url)、getRandomString(length, range)
    
*   getCurrentDate(format)、getTimeMinus(hours, minutes)、getTimeWithOffset(offsetUnit, offsetAmount)
    
*   unicodeToChinese(input)
    
*   validateLengthByRegex(...)
    

**示例**：生成附件文件名和去重键。

def fileUrl = page.xpath("//a\[contains(@href,'.pdf')\]/@href").get()  
fileUrl = ParseToolsUtils.resolve(page.getUrl().toString(), fileUrl)  
  
def fileName = ParseToolsUtils.getFileName(fileUrl)  
def dedupKey = ParseToolsUtils.addPrefix(fileName, "attachment:")  

**常见方法清单**：

*   addPrefix(String text, String prefix)
    
*   addPrefixAndSuffix(String text, String prefix, String suffix)
    
*   addSuffix(String text, String suffix)
    
*   aesDecrypt(String cipherText, String key)
    
*   batchReplaceString(String input, String fromValues, String toValues, String separator)
    
*   buqiHtml(String html)
    
*   completeHtml(String brokenHtml)
    
*   completeImageHtml(String htmlContent, String baseUrl, String imgAtr)
    
*   convertImgRelativeToAbsolute(String htmlContent, String baseUrl)
    
*   convertRelativeUrlsToAbsolute(Page page)
    
*   dateRangeCheck(String targetDate, Object startDate, Object endDate, Object trueValue, Object falseValue)
    
*   decrypt(String encryptedText, String key, String iv, String mode, String padding, String charset, String encoding)
    
*   extractAll(String input, String regex)
    
*   extractFirstGroup(String text, String regex)
    
*   extractJsonFromJsonp(String jsonp)
    
*   extractTextWithTrim(String text, String regex, int groupIndex)
    
*   formatPushTimeDateTime(String input, String inputFormat)
    
*   formatPushTimeDateTime(String dateString)
    
*   getBaseUrl(String url)
    
*   getByRegex(String text, String regex)
    
*   getCurrentDate(String format)
    
*   getFileName(String url)

### 5.2 ImageToolsUtils：处理图片地址和图片内嵌

适合“详情正文需要保留图片内容”的场景。

**常用方法**：

*   resolveImgUrl(pageUrl, imgSrc)：把相对图地址补成绝对地址
    
*   imgToBase64(pageUrl, imgSrc)：把单张图片转成 Base64
    
*   batchImgToBase64(pageUrl, originalHtml, imgSrcs)：批量把正文中的图片内嵌为 Base64
    
*   getMimeType(imgUrl)：根据后缀推断 MIME 类型
    

**示例**：先找出正文中的图片，再整体转成 Base64 HTML。

def contentHtml = page.xpath("//div\[@class='article-content'\]").get()  
def imgSrcs = page.xpath("//div\[@class='article-content'\]//img/@src").all()  
  
contentHtml = ImageToolsUtils.batchImgToBase64(  
    page.getUrl().toString(),  
    contentHtml,  
    imgSrcs  
)  

如果只想把图片地址补全，不想转 Base64，也可以单独使用：

def imgSrc = page.xpath("//div\[@class='article-content'\]//img\[1\]/@src").get()  
def absUrl = ImageToolsUtils.resolveImgUrl(page.getUrl().toString(), imgSrc)  

**注意**：

*   如果目标系统允许保留外链图片，优先只做相对地址补全，性能会更好。
    
*   只有确实需要“正文自带图片内容”时，再考虑转 Base64。
    

### 5.3 LogUtils：在脚本里打印可追踪日志

LogUtils 是排查脚本问题最直接的工具，尤其适合这些场景：

*   想确认 XPath / 正则是否提到了值
    
*   想知道某个分支为什么跳过
    
*   想记录关键请求参数、页码、文章数
    

**可用方法**：

*   debug(req, pattern, Object... args)
    
*   info(req, pattern, Object... args)
    
*   warn(req, pattern, Object... args)
    
*   error(req, pattern, Object... args)
    

**示例**：

LogUtils.info(request, "开始解析页面: {}", page.getUrl())  
LogUtils.debug(request, "列表页提取到 {} 条链接", linkList.size())  
  
if (!title) {  
    LogUtils.warn(request, "标题为空，页面: {}", page.getUrl())  
}  

**说明**：

*   第一个参数传 ParsePageRequest request，这样日志能关联到任务实例。
    
*   日志格式支持 {} 占位符，写法和 Slf4j 一致。
    
*   warn / error 适合记录异常数据，不建议把正常流程全打成高等级日志。
    

### 5.4 SpiderManagerUtils：请求构造与反爬协作工具

SpiderManagerUtils 不是“页面字段提取工具”，而是“请求处理工具”。它主要解决 4 类问题：

*   组装 POST 请求体
    
*   标准化参数，保证签名、去重、调试时格式稳定
    
*   判断 Cookie 是否属于当前请求域
    
*   与内部 Python 反爬服务对接
    

如果脚本只是“拿页面 -> 提字段”，通常不会频繁直接调用这个类；当你需要定制请求、模拟表单、处理反爬链路时，它就很重要。

#### 1）handlePostFormRequest

**作用**：把 Map<String, Object> 转成 application/x-www-form-urlencoded 请求体，并自动：

*   设置 request.method = POST
    
*   写入 requestBody
    
*   写入 GroovyConstants.POST\_PARAMS
    
*   设置 Content-Type: application/x-www-form-urlencoded
    

**示例**：构造表单请求。

import us.codecraft.webmagic.Request  
  
def req = new Request("https://example.com/search")  
def params = \[  
    keyword : "招标公告",  
    pageNum : 1,  
    category: "news"  
\]  
  
SpiderManagerUtils.handlePostFormRequest(req, params)  

**适用场景**：

*   站点搜索接口是传统表单提交
    
*   翻页参数、筛选参数需要放在 body 里
    
*   想把原始表单参数同步保留到 Request.extra
    

#### 2）handlePostJSonRequest

**作用**：把参数转成 application/json 请求体，并自动设置 POST 方法与请求头。

**示例**：构造 JSON 接口请求。

def req = new Request("https://example.com/api/list")  
def params = \[  
    pageNo   : 1,  
    pageSize : 20,  
    channelId: "notice"  
\]  
  
SpiderManagerUtils.handlePostJSonRequest(req, params)  

**补充说明**：

*   这个方法内部会把参数原样序列化成 JSON。
    
*   如果接口签名依赖“参数顺序稳定”，可以先用 normalizeJsonParams(params) 生成标准字符串做签名，再把原始参数交给该方法发请求。
    

#### 3）normalizeFormParams / normalizeJsonParams

这两个方法不直接发请求，而是把参数整理成稳定格式，适合做：

*   请求签名
    
*   去重键生成
    
*   调试时打印标准参数串
    

**示例**：生成稳定签名原文。

def params = \[b: "2", a: "1"\]  
  
def formText = SpiderManagerUtils.normalizeFormParams(params)  
def jsonText = SpiderManagerUtils.normalizeJsonParams(params)  
  
LogUtils.debug(request, "FORM参数串: {}", formText)  
LogUtils.debug(request, "JSON参数串: {}", jsonText)  

其中：

*   normalizeFormParams 会排序并做 URL 编码
    
*   normalizeJsonParams 会按 key 排序后再输出 JSON
    

#### 4）convertToValueList

**作用**：把不同形态的参数值统一转成字符串列表，支持：

*   Collection
    
*   数组
    
*   包含 #### 分隔符的字符串
    
*   普通单值
    

**示例**：

assert SpiderManagerUtils.convertToValueList(\["a", "b"\]) == \["a", "b"\]  
assert SpiderManagerUtils.convertToValueList(\["a", "b"\] as String\[\]) == \["a", "b"\]  
assert SpiderManagerUtils.convertToValueList("a####b####c") == \["a", "b", "c"\]  
assert SpiderManagerUtils.convertToValueList("single") == \["single"\]  

这个方法常用于“一个字段可能是单值，也可能是多值”的参数兼容处理。

#### 5）isCookieAllowed

**作用**：判断某个 Cookie 的 Domain 是否允许发给当前请求域名。

**示例**：

def allowed1 = SpiderManagerUtils.isCookieAllowed(".example.com", "news.example.com")  
def allowed2 = SpiderManagerUtils.isCookieAllowed("api.example.com", "api.example.com")  
def allowed3 = SpiderManagerUtils.isCookieAllowed(".other.com", "api.example.com")  

判断结果通常是：

*   父域 -> 子域：允许
    
*   完全相同域名：允许
    
*   无关域名：不允许
    

这个方法适合在脚本里做 Cookie 过滤、调试 Cookie 丢失问题。

#### 6）handleInnerPythonAntiRequest

这是 SpiderManagerUtils 里最“偏框架”的方法，用于和内部 Python 反爬服务联动。它会：

*   把 AntiPageRequest.paramsMap 发送到内部反爬服务
    
*   从返回结果里提取请求头并写入当前 Request
    
*   在需要时把代理写到 GroovyConstants.FANPA\_PROXY
    
*   把反爬服务返回的额外参数透传到 Request.extra
    

这一类能力通常不是普通解析脚本直接从零调用，而是用于“站点确实需要额外反爬链路”时，由脚本和平台配合使用。可以把它理解成“帮你把反爬服务返回的 headers / proxy 注入到请求里”。

如果只是写普通资讯站解析脚本，这个方法可以先了解用途，不一定需要直接使用。

**方法清单**：

*   convertToValueList(Object value)：将任意 value 转为字符串列表
    
*   handleInnerPythonAntiRequest(AntiPageRequest antiRequest, Request request, RestTemplate restTemplate, String innerUrl)：处理内部 Python 反爬调用
    
*   handlePostFormRequest(Request request, Map<String, Object> params)：处理 application/x-www-form-urlencoded 请求
    
*   handlePostJSonRequest(Request request, Map<String, Object> params)：处理 application/json 请求
    
*   isCookieAllowed(String cookieDomain, String requestHost)：判断 Cookie 是否属于当前请求域
    
*   normalizeFormParams(Map<String, Object> params)：标准化 FORM 参数
    
*   normalizeJsonParams(Map<String, Object> params)：标准化 JSON 参数
    

### 5.5 XPathContainsReplacer：处理 contains(text(), ...) 类场景

有些页面里，字段名和字段值混在一起，直接写 XPath 很容易拿不准。这时可以借助 XPathContainsReplacer。

**常用方法**：

*   replaceXPathContains(doc, element, containsText)：辅助处理类似 //span\[contains(text(),'发布时间')\]
    
*   extractFirstTextByContains(doc, keyword)：按关键字抓第一个文本
    
*   extractComplexText(doc)、extractSource(doc, keyword)：适合复杂文本块提取
    

**示例**：页面是“发布时间：2026-03-31 09:00”这种结构时，可以先按关键字抓文本，再二次清洗。

def doc = org.jsoup.Jsoup.parse(page.getRawText())  
def rawText = XPathContainsReplacer.extractFirstTextByContains(doc, "发布时间")  
def pushTime = ParseToolsUtils.extractFirstGroup(rawText ?: "", "发布时间\[:：\]?\\s\*(.\*)")  
pushTime = ParseToolsUtils.normalizeDateTime(pushTime)  

**适用建议**：

*   标签文本和字段值紧挨在一起时，这个类往往比手写复杂 XPath 更省事。
    
*   如果普通 XPath 已经能稳定取到值，就优先用普通 XPath，脚本会更直观。
    

**方法清单**：

*   extractComplexText(Document doc)
    
*   extractFirstTextByContains(Document doc, String keyword)
    
*   extractSource(Document doc, String keyword)
    
*   replaceXPathContains(Document doc, String element, String containsText)
